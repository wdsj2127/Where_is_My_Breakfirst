# Where is My Breakfirst - 代码注释规范

## 文件头注释

每个 `.gd` 脚本文件开头必须包含文件头注释块，格式如下：

```gdscript
# ============================================================
# 文件名.gd - 简短描述
# ============================================================
# 功能：详细描述该脚本实现的功能
# 方法：
#   方法名(参数类型) -> 返回类型  - 方法简述
#   ...
# ============================================================
```

### 规则

1. **分隔线**：用 `# =====...=====` 包裹，宽度60个字符
2. **文件名**：与实际文件名一致
3. **功能**：用一两句话概括脚本的核心职责
4. **方法列表**：列出所有自定义方法（不含 `_ready`/`_process` 等引擎回调的隐式方法，但如果重写了也列出），格式为 `方法名(参数) -> 返回类型 - 一句话描述`
5. 方法列表按定义顺序排列

## 方法注释

每个自定义方法前必须添加注释，说明作用、参数、返回值：

```gdscript
## 方法作用的一句话描述
## 参数 param1: 参数1的说明
## 参数 param2: 参数2的说明
## 返回: 返回值说明
func my_method(param1: int, param2: float) -> bool:
```

### 规则

1. 使用 `##`（GDScript文档注释）而非 `#`
2. **作用**：第一行，简述方法做什么
3. **参数**：每个参数一行，格式 `参数 名字: 说明`；如果参数少且简单，可合并为一行
4. **返回**：格式 `返回: 说明`；void方法省略
5. 无参数无返回值的简单方法可只写作用行
6. 引擎回调（`_ready`/`_process`/`_physics_process`）也需注释

## 示例

```gdscript
# ============================================================
# example.gd - 示例脚本
# ============================================================
# 功能：演示注释规范
# 方法：
#   _ready()              - 初始化
#   calculate(a, b) -> int - 计算两数之和
# ============================================================

extends Node

## 初始化：打印启动信息
func _ready() -> void:
	print("ready")

## 计算两个整数的和
## 参数 a: 第一个整数
## 参数 b: 第二个整数
## 返回: a + b 的结果
func calculate(a: int, b: int) -> int:
	return a + b
```
